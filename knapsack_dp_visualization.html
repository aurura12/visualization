观察二维表如何逐格填充，并回溯得到最优物品组合。

说明

0/1 背包问题：给定容量为 C 的背包和 n 件物品，每件物品有重量 wi 和价值 vi，如何选择装入的物品使总价值最大？

动态规划的核心思想：将原问题拆解为规模更小的子问题，通过记录子问题的最优解来避免重复计算。定义 V[i][j] 表示"在前 i 件物品中挑选，装入容量为 j 的背包，能获得的最大价值"。那么 V[i][j] 的递推关系为：

V[i][j] = V[i‑1][j]（容量不足，无法装入第 i 件物品）
V[i][j] = max(V[i‑1][j], V[i‑1][j‑wi] + vi)（容量充足，选择装或不装中更优者）

表格的行对应物品（行 1～5 的权重/价值已在表头标注），列对应背包容量 0～10。填表从第 1 行开始逐行逐列进行，每一步的决策只依赖上一行的结果——这就是"动态规划"中"状态转移"的直观体现。最终 V[5][10] 就是全局最优解（本示例为 15）。

本页面使用容量 C=10、5 件物品的重量为 {2, 2, 6, 5, 4}、价值为 {6, 3, 5, 4, 6} 作为示例。页面会逐格演示填表过程：黄色高亮当前正在计算的格子；橙色标记「不装入」的参考来源（V[i-1][j]）；紫色标记「装入」的参考来源（V[i-1][j-w] + v）；绿色格子表示回溯路径上被选中的物品；蓝色格子表示容量不足直接继承上一行。侧面板同步显示当前决策依据和对应的 Python 代码行高亮。
